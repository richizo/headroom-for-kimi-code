# Task 03-03 — Testes de backend Moonshot para thinking

## Arquivos
- `tests/test_backends/test_moonshot.py` (criar se não existir)

## Objetivo
Validar o comportamento do backend Moonshot com thinking habilitado/desabilitado e sem thinking.

## Casos de teste
1. `test_moonshot_thinking_enabled_returns_reasoning_content`
   - Request contém `"thinking": {"type": "enabled"}`.
   - Mock upstream retorna fixture thinking.
   - Verifica que response contém `reasoning_content`.

2. `test_moonshot_thinking_disabled_does_not_return_reasoning_content`
   - Request contém `"thinking": {"type": "disabled"}`.
   - Mock upstream retorna fixture instant.
   - Verifica que `reasoning_content` não está presente.

3. `test_moonshot_instant_mode_no_thinking_parameter`
   - Request não envia `thinking`.
   - Mock upstream retorna fixture instant.
   - Verifica resposta padrão.

4. `test_moonshot_thinking_parameter_forwarded_to_upstream`
   - Captura o body enviado ao mock e verifica que `thinking.type` está presente.

## Setup
- Usar `responses` ou `httpx.MockTransport` conforme padrão do projeto.
- Carregar fixtures via helper existente (`tests.fixtures.load_json` se houver).

## Critério de sucesso
- Todos os testes passam isoladamente e em conjunto.
- Cobertura das linhas de request/response do backend Moonshot.
