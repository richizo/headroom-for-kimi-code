# Task 03-05 — Documentação de uso do thinking Moonshot

## Arquivos
- `wiki/moonshot.md` (criar ou atualizar)
- `CHANGELOG.md` (ou `.changelog.md` se for o padrão do projeto)

## Objetivo
Documentar para usuários como habilitar/desabilitar thinking e onde encontrar `reasoning_content`.

## Conteúdo
1. Modelos que suportam thinking.
2. Exemplo de request com thinking habilitado:
   ```json
   {
     "model": "kimi-k2.6",
     "messages": [...],
     "thinking": { "type": "enabled", "keep": "all" }
   }
   ```
3. Exemplo de request com thinking desabilitado (modo instant):
   ```json
   {
     "model": "kimi-k2.6",
     "messages": [...],
     "thinking": { "type": "disabled" }
   }
   ```
4. Exemplo de resposta com `reasoning_content`.
5. Nota sobre modelos que não permitem desabilitar thinking.

## Critério de sucesso
- Documentação clara e com exemplos válidos.
- Entrada no changelog descrevendo o suporte a thinking/reasoning.
