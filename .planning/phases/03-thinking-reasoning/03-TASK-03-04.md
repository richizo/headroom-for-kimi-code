# Task 03-04 — Teste de integração proxy ↔ Moonshot

## Arquivos
- `tests/integrations/test_proxy_moonshot.py` (ou padrão equivalente)

## Objetivo
Validar que `reasoning_content` percorre todo o caminho: cliente → proxy → backend MoonShot (mock) → cliente.

## Casos de teste
1. `test_proxy_moonshot_thinking_end_to_end`
   - Cliente envia POST `/v1/chat/completions` com `"thinking": {"type": "enabled"}`.
   - Mock da upstream Moonshot retorna fixture thinking.
   - Verifica status 200 e `reasoning_content` no JSON de resposta.

2. `test_proxy_moonshot_instant_end_to_end`
   - Cliente envia POST sem `thinking`.
   - Mock retorna fixture instant.
   - Verifica que `reasoning_content` não aparece.

## Setup
- Usar `TestClient` do FastAPI/Starlette se o proxy for ASGI.
- Mock da upstream com `responses`/`httpx`.

## Critério de sucesso
- Testes passam.
- `reasoning_content` é idêntico ao fixture no corpo da resposta final.
