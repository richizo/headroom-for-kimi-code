# Task 03-01 — Verificar pass-through no backend Moonshot

## Arquivos
- `headroom/backends/moonshot_backend.py`
- `headroom/backends/base.py` (se houver parsing genérico de resposta)

## Objetivo
Garantir que o backend Moonshot encaminhe o parâmetro `thinking` da request para a upstream e preserve o campo `reasoning_content` da response.

## Passos
1. Ler `headroom/backends/moonshot_backend.py`.
2. Identificar o método que monta/envia a request (tipicamente `send_openai_message` ou similar).
3. Verificar se há lista explícita de campos permitidos no request body.
4. Verificar se há lista explícita de campos extraídos da response.
5. Se `thinking` for removido ou `reasoning_content` for ignorado, ajustar para preservar.
6. Se não houver filtro, adicionar comentário/docstring indicando suporte a thinking.
7. Executar testes relevantes do backend.

## Critério de sucesso
- Request body contendo `thinking` chega à upstream intacto.
- Response body contendo `reasoning_content` é retornado ao proxy intacto.
- Nenhum teste existente quebra.

## Dependências
- Nenhuma (não requer modelos reais; usar mocks).
