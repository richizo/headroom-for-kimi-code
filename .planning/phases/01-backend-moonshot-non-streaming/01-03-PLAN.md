       ```python
       MoonshotBackendType: Any = None

       def _load_moonshot_backend() -> Any:
           global MoonshotBackendType
           if MoonshotBackendType is None:
               from headroom.backends import MoonshotBackend
               MoonshotBackendType = MoonshotBackend
           return MoonshotBackendType
       ```

    2. In `create_proxy_backend`, add a branch before the LiteLLM fallback logic:
       ```python
       if backend == "moonshot":
           try:
               backend_cls = moonshot_backend_cls or _load_moonshot_backend()
               api_key = os.environ.get("MOONSHOT_API_KEY")
               base_url = os.environ.get("MOONSHOT_BASE_URL")
               instance = cast(
                   "Backend",
                   backend_cls(
                       api_key=api_key,
                       base_url=base_url or None,
                   ),
               )
               logger.info("Moonshot backend enabled")
               return instance
           except ImportError as exc:
               logger.warning("Moonshot backend not available: %s", exc)
               return None
           except Exception as exc:
               logger.error("Failed to initialize Moonshot backend: %s", exc)
               return None
       ```

    3. Add `moonshot_backend_cls: Any | None = None` to the `create_proxy_backend` signature.

    4. In `format_backend_status`, add a branch for `backend == "moonshot"` returning `"Moonshot"`.
  </action>

  <action>
    Create `tests/test_backends/test_moonshot.py` with the following test cases:

    1. `test_name` — asserts `MoonshotBackend(...).name == "moonshot"`

    2. `test_requires_api_key` — asserts `ValueError` when no API key is provided and env var is unset

    3. `test_api_key_from_argument` — asserts `backend.api_key == "arg-key"`

    4. `test_api_key_from_env` — set `MOONSHOT_API_KEY=env-key`, instantiate without arg, assert `backend.api_key == "env-key"`

    5. `test_default_base_url` — asserts `backend.base_url == "https://api.moonshot.ai/v1"`

    6. `test_custom_base_url` — instantiate with `base_url="https://api.moonshot.cn/v1"`, assert trailing slash is stripped

    7. `test_supports_model_kimi` — parametrize over `kimi-k2`, `kimi-latest`, `kimi-k2-0711-preview`; assert True

    8. `test_supports_model_rejects_others` — assert False for `gpt-4`, `claude-3-opus`

    9. `test_map_model_id_passes_through` — assert `map_model_id("kimi-k2") == "kimi-k2"`

    10. `test_send_openai_message_uses_profile_api_key` — use `respx` or `httpx` transport mock to intercept POST to `https://api.moonshot.ai/v1/chat/completions`. Verify request header `Authorization` is `Bearer profile-key` even if client headers contain a different key. Return a mock OpenAI chat completion response and assert the backend returns it.

    11. `test_send_openai_message_resolves_kimi_latest` — mock `/v1/models` to return `{"data": [{"id": "kimi-latest"}, {"id": "kimi-k2"}]}`. Send request with model `kimi-latest` and assert upstream request body has model `kimi-latest` (since Moonshot accepts the alias as a model ID). Alternatively, if the mock returns only `kimi-k2`, assert fallback behavior. Keep this test simple and deterministic.

    12. `test_send_openai_message_passthrough_error` — mock upstream returning 400 with `{"error": {"message": "invalid temperature", "type": "invalid_request_error"}}`. Assert backend returns `BackendResponse` with status_code 400 and same body.

    Use `pytest.mark.asyncio` for async tests. If `respx` is not available, use `httpx.MockTransport` or monkeypatch `backend._client`.
